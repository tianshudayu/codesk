# 手机端操作说明

## 适用范围

这份说明对应 `E:\codex-mcp-mobile` 当前的 MCP 线原型，用于：

- 查看本机 Codex 线程库
- 查看运行中会话的实时进度
- 创建新线程
- 继续发送需求
- 处理审批

不用于远程点击桌面窗口，不替代 [E:\远程桌面](E:/远程桌面) GUI 远控线。

## 连接方式

### 方式一：同 Wi‑Fi 直连

1. 电脑启动 bridge：`python -m bridge.main`
2. 手机打开：`http://<电脑局域网IP>:8890/`
3. 在电脑本机打开：`http://127.0.0.1:8890/api/admin/state`
4. 读取其中的 `pairCode`
5. 手机输入配对码完成连接

### 方式二：relay 扫码

1. 电脑启动 relay：`python -m mcp_relay.main`
2. 电脑启动 bridge：`python -m bridge.main`
3. 在电脑本机打开：`http://127.0.0.1:8890/api/admin/state`
4. 查看 `relay.mobileUrl` 或对应二维码
5. 手机打开该链接
6. 输入本机显示的 `pairCode`

## 手机端主要区域

### 线程库

- 显示白名单工作区内的历史线程
- 可查看线程标题、预览、来源、状态
- 点开后会显示历史 turns / items
- 点击“恢复到手机端”后，该线程会进入手机可继续操作的会话区

### 运行中会话

- 显示手机新建或恢复后的会话
- 可查看当前状态、摘要、错误信息
- 支持继续发送需求
- 支持取消任务

### 进度与审批

会显示这些事件：

- `message.delta`
- `message.completed`
- `tool.started`
- `tool.completed`
- `turn.plan.updated`
- `turn.diff.updated`
- `command.output.delta`
- `filechange.output.delta`
- `session.completed`
- `session.failed`
- `approval.required`
- `approval.resolved`

## 审批处理

### 命令 / 文件 / 权限审批

默认可选：

- `approve`
- `approve_session`
- `reject`
- `cancel`

### request_user_input

1. 在审批卡片输入答案
2. 点击 `submit`
3. Bridge 会把答案回传给 Codex

## 推荐工作流

### 跟进已有项目线程

1. 打开手机端
2. 在“线程库”里选择目标线程
3. 查看历史 turns 和最近状态
4. 点击“恢复到手机端”
5. 在“运行中会话”里继续发送新的需求

### 新建一个手机发起的任务

1. 选择工作区
2. 输入任务描述
3. 点击“新建线程”
4. 在右侧实时查看计划、工具、输出和 diff
5. 如果出现审批，在手机端直接处理

## 常见问题

### 1. 看不到线程

- 检查 `CODEX_MCP_WORKSPACES`
- 线程必须属于白名单工作区

### 2. 后端不可用

先打开：

```text
http://127.0.0.1:8890/api/admin/state
```

检查：

- `backendAvailable`
- `backendLastError`
- `backendSuggestedFix`

### 3. relay 扫码后没有进度

检查：

- relay 是否启动
- bridge 是否启动
- `relay.status` 是否为 `connected`
- 手机是否已完成配对

### 4. 审批按钮点了没反应

- 确认当前会话不是断线状态
- 刷新“运行中会话”
- 查看最近事件里是否出现 `approval.resolved`
